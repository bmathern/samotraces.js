Introduction

In the first part of the tutorial, you have seen how to create a first basic visualisation of a trace with the {@link Samotraces.UI.Widgets.d3Basic.TraceDisplayIcons|TraceDisplayIcons} widget. In the second part of the tutorial, you have learnt how to add other widgets. In this third and last part of the tutorial, you will learn how to tune the visualisation of a trace to more specific needs.

In the previous part of the tutorial, when clicking on the different obsels, you might have noticed that the trace you have loaded looks very similar to music notes (with french notation!). If you have a closer look at the trace.csv file, you will see that the obsels type correspond to the note's height (do, re, mi, etc.) and each obsel has an attribute "duration" which correspond to the note's duration ("blanche", "noire", etc.).

For non-french musicians: a "blanche" is a "minim" (amercian: "half note") and a "noire" is a "crotchet" (american: "quarter note") and the note's heights are:

  • C for "do"
  • D for "re"
  • E for "mi"
  • F for "fa"
  • G for "sol"
  • A for "la"
  • B for "si"

In this tutorial, you will display these notes on a music sheet. For this purpose, you will setup the {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} widget, and tweak the HTML and CSS files.

Reminder from part 2 of the tutorial.

Let's start with the code obtained at the end of the previous part of the tutorial: 

<!doctype html>
<html>
	<head>
		<meta charset="utf-8">
		<script type="text/javascript" src="javascript/d3.js"></script>
		<script type="text/javascript" src="javascript/jquery.js"></script>
		<script type="text/javascript" src="javascript/jquery.mousewheel.js"></script>
		<script type="text/javascript" src="javascript/Samotraces.js"></script>
		<link rel="stylesheet" type="text/css" href="css/samotraces.css"> 
	</head>
	<body>
		<div id="my-widget"></div>
		<div id="scale"></div>
		<div id="importer"></div>
		<div id="obsel-inspector"></div>
		<script type="text/javascript">
		function init() {
			// Create logical objects
			var trace 		= new Samotraces.LocalTrace();
			var tw			= new Samotraces.TimeWindow({start: 0,end: 20});
			var obsel_selector	= new Samotraces.Selector('Obsel');

			// Create widgets
			new Samotraces.UI.Widgets.TraceDisplayIcons('my-widget',trace,tw);
			new Samotraces.UI.Widgets.WindowScale('scale',tw);
			new Samotraces.UI.Widgets.ImportTrace('importer',trace);
			new Samotraces.UI.Widgets.ObselInspector('obsel-inspector',obsel_selector);

			// Select Obsel when clicked
			callback = function(event) {
				var obsel = $.data(event.target,'Samotraces-data');
				obsel_selector.select(obsel);
			};
			$('body').on("click",".Samotraces-obsel",callback);
		}

// calling the init function when the DOM has been loaded
window.addEventListener('DOMContentLoaded', init );
		</script>
	</body>
</html>

Reminders

  • This files "Samotraces.js", "jquery.js", "jquery.mousewheel.js", and "d3.js" must be placed in a folder called "javascript" and the file "samotraces.css" in a folder called "css".
  • This code contains <div> HTML tag elements, each having an unique id attribute. Each widget will be instanciated in the corresponding HTML DIV element.
  • The Javascript init() function is run when the page is loaded. This function will:
    • Create a trace object, an obsel selector object and a time window object.
    • Create the following widgets:
      • {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons}: a trace visualisation widget
      • {@link Samotraces.UI.Widgets.WindowScale|WindowScale}: a time scale widget
      • {@link Samotraces.UI.Widgets.ImportTrace|ImportTrace}: a widget to import CSV files in the current trace
      • {@link Samotraces.UI.Widgets.ObselInspector|ObselInspector}: a widget that inspect obsels

Setting up the visualisation: displaying the image you want

First, you will have to download the (four) following images that will be used in our visualisation (Right click -> save image as) and place them to a new "images" folder of your project:

Now, that you have the relevant images ready, we will use what has already been done in the previous parts of the tutorial and change the visualisation so that music notes are displayed instead of the default question-mark symbol.

In order to change the default visualisation symbol of the displayed trace, we will go back to the documentation of the {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} widget.

When you initialise the {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} widget the last parameter is an optional parameter. This parameter corresponds to the visualisation parameters. More precisely, this parameter is a {@link Samotraces.UI.Widgets.TraceDisplayIcons.VisuConfig|VisuConfig} object, which defines how each obsel will be displayed: the image name (url), its height, width, and its coordinates x and y. Let's have a look at each parameters more precisely:

  • x: the X coordinates of the top-left corner of the image
  • y: the Y coordinates of the top-left corner of the image
  • width: the width (in pixels) of the image
  • height: the height (in pixels) of the image
  • url: the URL of the image to display (default: a question-mark symbol)

The first think we want to do is to define the image that will be displayed for each obsel. Instead of displaying a question-mark symbol, it would be nicer to display a music note. This is done by editing the url field of the optional object.

As you have copied the images in the "images" folder, the URL of the image of a music note should be something like "images/noire.png". Let's try this code: 

options = {
	url: "images/noire.png"
};

As we add this 4th optional parameter to instanciate the {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} widget, the code should now look like this: 

// Visualisation options
options = {
	url: "images/noire.png"
};

// Create widgets
new Samotraces.UI.Widgets.TraceDisplayIcons('my-widget',trace,tw,options);

When trying it, you should see something like this:

This is good. This is more music like. It is a bit too small though.

The original image file is 34 pixel in width by 60 pixels in height. So, why is it so small? Well, this is because the {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} widget displays images of 16px by 16px by default. So, now, you want to edit the visualisation to change de default width and default height. This is pretty easy (don't forget the commas to separate the attributes!): 

options = {
	url: "images/noire.png",
	width: 34,
	height: 60
};

Uh Oh... The image doesn't fit. Let's try to change the y coordinate at which it is diplayed (default value is 17px from the top of the widget): 

options = {
	url: "images/noire.png",
	width: 34,
	height: 60,
	y: 2
};

Ok, this doesn't work. The widget is too narrow. Well, don't worry, there is a way to make the widget wider. Good news: for this, you don't need to write any JavaScript! You'll have to edit the CSS properties of the widget.

CSS and Samotraces.js

A short introduction to CSS

If you don't know what CSS are, here is a very short introduction. CSS stands for Cascading Style Sheets. CSS is a language used to define styles that you can apply to HTML content. With CSS, you can typically define the height of an element, or its background colour, etc.

The goal of this tutorial it not to teach you about CSS. However, there are two concepts that you need to understand: the concept of ID and the concept of class.

Each ID is unique in one HTML page. You have already defined IDs in the HTML content you have writtend so far. An ID is stored in the "id" attribute of any HTML element. One HTML element can have at most one ID. The ID will identify this particular HTML element.

A class, on the other hand, can be shared by several HTML elements. One HTML element can have one or several classes. Classes are stored in the "class" attribute of any HTML element. If an HTML element has several classes, they are separated in the HTML code by spaces

In CSS language, IDs are represented with the prefix "#" and classes are represented by the prefix ".".

What you should know about CSS in Samotraces.js

Samotraces.js uses CSS to correctly display the widgets. That is why, from the very begining of the tutorial you had to include a CSS file (samotraces.css). If you were to try to remove this file, you would see that the display is arbitrary and doesn't work the way you want.

The samotraces.css file is just a default template to display Samotraces.js widgets. But... How does it work? How does the CSS file know which style is applied to which HTML element? So far, the HTML element containing the widgets only have (arbitrary) IDs. (Note that if you change the IDs in the HTML file and update the JavaScript code, the visualisation will still work). Does the CSS file communicate with the JavaScript code to know which style to apply to which plugin?

Well, the solution is simple and based on the concept of classes. Whenever a widget of Samotraces.js is instanciated, the corresponding HTML element will be given two classes. The classes are generated by the Javascript code of each Widget. The first class is the class "Widget". The second class is the class corresponding to the specific widget you have been instanciating (for instance "Widget-TraceDisplayIcons"). Then, if you look at the CSS file samotraces.css, you will see that styles are defined for each widget. The common class "Widget" is shared by all widgets and is used to define common set of attributes that are shared by all Widgets.

Of course, the samotraces.css file is just a default template to display Samotraces.js widgets. Feel free to edit it to make your own templates!

Editing the CSS of a widget

Now that we know how it works, let's go back to our initial problem. We want to change the height of the {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} widget. For this, we must edit the CSS style of the "Widget-TraceDisplayIcons" class.

There are several ways to edit the CSS styles. You can either edit your own template and use the samotraces.css file as a basis. You can also create your own css file which will complement the currently existing samotraces.css file. Or you can add the style directly in the HTML code with a <style> tag. In real life example, the first two options are highly recommanded as they make the code far more maintanable and easy to read. However, for the purpose of the tutorial, we will use the last method, which will be easier to follow if you are not used to CSS: we will keep editing the same file that contains the HTML and the Javascript code. (Note that if you are willing to write your own CSS file or edit the existing CSS file, you will have to update the paths to the background images.)

If you look at the samotraces.css file, you will see that {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} widget's height is setup to 50px. This explains why our 60px image wouldn't fit. Now, we will write some CSS code in the HTML page that will override this property to make the widget 150px in height: 

<style type="text/css">
div.Widget-TraceDisplayIcons {
	height: 150px;
}
</style>

Now, if you try again. The notes will actually fit in the widget. Pheeew!

So, why not going further now and add a little background representing the music staff ("port√©e" in french)? For instance, we could put the staff as a background image of the widget. You can do it this way: 

div.Widget-TraceDisplayIcons {
	height: 150px;
	background: url("./images/portee.png") repeat-x 0px 40px;
}

Note that the "repeat-x" argument means that the image of the staff is repeated horizontaly, and "0px 40px" means that an offset of 0px horizonaly and 40px vertically is applied to display the image. The offset is used to display the staff somewhat in the middle of the widget, instead of the very top.

So, now, you have a staff and notes, but notes doesn't really match the staff. Ideally, we would like to have each note (A, B, C, D, etc.) at their correct spot on the staff.

But hang on, how do I know if a note is A, B, C, etc.? Well, this information is stored on the {@link Samotraces.Obsel|Obsel} object. {@link Samotraces.Obsel|Obsel} objects each have a type, a timestamp and optional attributes. In our example, the note's height (A, B, C, etc.) is stored in the type of the obsel.

So now, we have to go back to the options JavaScript object that is used by {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} widget to adapt the visualisation of each obsel: 

options = {
	url: "images/noire.png",
	width: 34,
	height: 60,
	y: 2
};

But hang on, we don't want all obsels to be displayed at the same y. We want each obsel to be displayed to it's own y, corresponding to its own type. How do we do that? Well, if you go back to the documentation of the {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} widget, you will see that each option argument can be either a value (like what we have done so far), or a function. If it is a function, the function will be passed as argument the specific {@link Samotraces.Obsel|Obsel} object to display and should return the calculated value for this obsel.

Based on that, we need to write a function that will calculate the y value depending on the obsel type. Here is what it might look like (I have calculated the values for you): 

options = {
	url: "images/noire.png",
	width: 34,
	height: 60,
	y: function(obsel) {
		switch(obsel.type) {
			case "do":
				return 74;
			case "re":
				return 66.5;
			case "mi":
				return 59;
			case "fa":
				return 51.5;
			case "sol":
				return 44;
			case "la":
				return 36.5;
			case "si":
				return 29;
			default:
				return 0;
		}
	}
};

So, the y field is now a function that takes an obsel as an argument. It looks at the obsel type (obsel.type) to define the y parameter and returns it. For those who like more compact formulas, you could write it this way (which returns the very same result and is easier to maintain): 

	y: function(o) {
		var note_height = {'do':0, 're':1, 'mi':2, 'fa':3, 'sol':4, 'la':5, 'si':6};
		return 74-note_height[o.type]*7.5;
	}

The same way you calculated the y field, you can also calculate the url field. You don't want all the obsels to be displayed with the same image. For instance, in the trace you are currently visualising, not all obsels are crotchet. Some are minims. Let's write a function that does that: 

options = {
	url: function(obsel) {
		switch(obsel.attributes.duration) {
			case "noire":	// crotchet
				return "images/noire.png";
			case "blanche":	// minims
				return "images/blanche.png";
			default:
				return "";
		}
	},
	width: 34,
	height: 60,
	y: function(o) {
		var note_height = {'do':0, 're':1, 'mi':2, 'fa':3, 'sol':4, 'la':5, 'si':6};
		return 74-note_height[o.type]*7.5;
	}
};

The duration of the obsel-note is stored in the duration attribute. As duration is an optional attribute (the non-optional attributes are the id, the type and the timestamp), its value is stored in the "attributes" field of the {@link Samotraces.Obsel|Obsel} object. That explains why it is accessed with the code obsel.attributes.duration.

Now, you should have a much nicer visualisation (like the image below), that actually matches the note's heights and durations! Congratulation!

Again, a more compact function can be used to calculate the URL of the image: 

	url: function(o) {
		return 'images/'+o.attributes.duration+'.png';
	},

Polishing the result

This section is for the more advanced users and do not teach anything about Samotraces that have not been already stated. Feel free to skip and go directly to the final code.

In order to make it look a bit nicer, you might want to add the C clef. This will require to work only with the CSS and HTML content. Here is the code HTML code followed by the CSS code. The explanations comes along. 

<div id="clef"></div>
<div id="widgets">
    <div id="my-widget"></div>
    <div id="scale"></div>
	<div id="importer"></div>
	<div id="obsel-inspector"></div>
</div>

Two new HTML <div> have been created. One with the ID "clef", which will contain the image of the C clef, one with the ID "widgets" that acts like a container of all the widgets (note that this DIV element literally contains all the DIV elements of the widgets). The reason why all the widgets are encapsulated into a "widgets" DIV is to make it possible for all the widget to have the same width. Without doing so, the width of the {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} and {@link Samotraces.UI.Widgets.WindowScale|WindowScale} widget might be different, and therefore, the time representation would not match. 

div.Widget-WindowScale {
	width: auto;
}

div.Widget-TraceDisplayIcons {
	height: 150px;
	background: url("./images/portee.png") repeat-x 0px 40px;
	width: auto;
}

#clef {
	width: 62px;
	background: url("./images/clef.png") no-repeat 0px 41px;
	float: left;
	height: 300px;
}

#widgets {
	width: auto;
	height: 100%;
}

On the CSS side, the "clef" DIV element is setup to display the "clef.png" image as a background image. The width of the "widget" DIV element is setup to auto so that it adapts itself to the available space.

the attribute width of the widgets {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} and {@link Samotraces.UI.Widgets.WindowScale|WindowScale} had to be overridden. The default value in samotraces.css file is width: 100%;, which corresponds to the width of the HTML page, not to the width of the available space on the left.
